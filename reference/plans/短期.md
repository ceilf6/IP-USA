下面是**短期 1 个月试运行方案**。默认假设：你是个人使用 Claude Code，主要在一台电脑上用，不先折腾家庭路由器；预算接受约 `150-300 RMB/月` 级别；目标是降低网络侧风控，不做账号共享、API 中转、自动化滥用。

**方案选择**
选“独享静态美国住宅宽带 IP/SOCKS/VPS + 前置中转”：

```text
你的电脑
  -> 稳定前置节点/CN2/GIA/机场
  -> 美国住宅宽带 IP 落地
  -> Claude / Anthropic
```

短期不要买普通 VPS 当最终出口，也不要用 Claude API 中转站。中转站最大问题不是 IP，而是你的代码、Prompt、账号请求都过第三方服务器，存在日志、模型替换、账号污染风险。

**购买清单**

1. **美国住宅宽带出口**
   优先级：`VIRCS / AaIT / AaITR` 这类明确标注“独享静态住宅 IP、AT&T/Frontier/Spectrum 等 ISP、Socks5 或 VPS”的服务。
   先买月付，不年付。带宽 `30-50Mbps` 足够 Claude Code，没必要上 300Mbps。

2. **前置中转**
   如果你已有稳定机场，先用现有机场做前置。
   如果没有，再考虑 CN2/GIA VPS。短期不建议一开始就买路由器和家庭中枢。

3. **本机代理客户端**
   - macOS：Clash Verge Rev / Shadowrocket
   - Windows：v2rayN / Clash Verge Rev，建议开 TUN
   - 不建议一开始上 OpenClash 路由器方案，排障成本高。

**服务商验收标准**
买之前或开通后马上查：

- `ipinfo.io`：ASN/Org 应该是 `AT&T`、`Frontier`、`Comcast`、`Spectrum` 这类 ISP，不应是 `hosting`、`datacenter`。
- `scamalytics.com`：Fraud Score 尽量低，理想 `< 10`。
- `ipqualityscore.com`：不要出现 VPN / Proxy / TOR / Hosting 高风险标记。
- `ping0.cc` 或类似工具：看是否识别为家宽/原生/低风控。
- 如果卖家不给试、不给换 IP、无法说明 ASN，不买。

**配置原则**
Claude 相关流量走住宅 IP，其他低敏流量可以走普通节点。短期为了稳，最开始 3-7 天建议 Claude 使用时直接全局走住宅 IP，不急着搞复杂分流。

关键配置：

```yaml
ipv6: false
client-fingerprint: chrome
dns:
  enable: true
```

DNS 必须跟代理走，避免“出口在美国，DNS 在中国”的地域不一致。Windows 下要开 TUN，否则终端里的 Claude Code 可能不走代理。

**Claude 分流域名**
最少保证这些走住宅出口：

```text
claude.ai
api.anthropic.com
anthropic.com
console.anthropic.com
```

实际 Claude Code 还可能访问认证、CDN、遥测、更新相关域名，所以短期最稳做法是：**使用 Claude Code 时切到住宅 IP 全局模式**。确认稳定后再做细分流。

你的现有配置**算前置节点**。  
“机场”就是你买的那套 Clash 订阅；“节点”是里面一个个可选服务器；“前置节点”是链式代理里的第一跳，用来把你从国内稳定送到美国住宅 IP。Claude 最终看到的不是这个机房 IP，而应该是最后一跳的**美国住宅 IP**。

你的短期方案可以这样定：

**目标架构**

```text
macOS
 -> 你现有 Clash 机场节点，机房 IP，只做前置中转
 -> 新买的美国住宅 IP / SOCKS5 / VPS，只做最终出口
 -> Claude / Claude Code
```

**第一步：买住宅出口**

不要买动态轮换住宅代理，不要买共享代理池。

**第二步：客户端建议**
你现在用的是 Clash For Windows 的 macOS 版。它能用，但项目本体早已停止维护，链式代理能力也取决于内核版本。短期我建议你换成 **Clash Verge Rev**，原因是它基于 Mihomo，支持 `dialer-proxy` 链式代理；Mihomo 官方文档明确说明 `dialer-proxy` 可以让一个代理通过另一个代理建立连接。

保守做法：

```text
下载 Clash Verge Rev
导入你现在的机场订阅
再手动添加住宅 SOCKS5 节点
用 dialer-proxy 把住宅节点挂到机场节点后面
```

**第三步：配置逻辑**
假设你买到的住宅 IP 是 SOCKS5，服务商给你：

```text
server: 1.2.3.4
port: 12345
username: user
password: pass
```

在配置里增加一个住宅节点，核心是 `dialer-proxy`：

```yaml
proxies:
  - name: "US-Residential"
    type: socks5
    server: 1.2.3.4
    port: 12345
    username: user
    password: pass
    udp: false
    dialer-proxy: "你的机场节点名"
```

这里的 `"你的机场节点名"` 必须和 Clash 里某个可用机场节点名称完全一致。  
流量路径会变成：

```text
你的 Mac -> 机场节点 -> US-Residential -> Claude
```

**第四步：先全局，不先分流**
前 3-7 天不要搞复杂规则。使用 Claude Code 时直接选择 `US-Residential` 并开启全局/TUN，确保浏览器和终端都走同一个住宅出口。

macOS 验证：

```bash
curl https://ipinfo.io
curl https://ifconfig.me
```

看到的必须是住宅 IP，不是你机场的机房 IP，也不是国内 IP。

还要检查：

```text
IPv6：关闭
DNS：走代理
WebRTC：无真实 IP 泄露
Claude Code 终端请求：和浏览器出口一致
```

**第五步：使用节奏**
第一天只轻量使用：

```text
登录 Claude
少量聊天
少量 Claude Code
不要同时多设备
不要频繁切节点
不要一上来跑多个大任务
```

稳定 2-3 天后再正常用。一个月试用期间只观察三件事：Claude 是否稳定、IP 检测是否干净、晚高峰是否断流。
